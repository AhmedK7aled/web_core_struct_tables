١١. Legacy vs Modern – المقارنة الشاملة

كل خاصية قديمة ومقابلها الحديث في CSS

📌
الخصائص الملغاة لا تزال تعمل في المتصفحات، لكنها ليست جزءاً من معيار HTML5 الرسمي. استخدامها في مشاريع جديدة يُعتبر ممارسة سيئة ويُصعّب الصيانة. الاستثناء الوحيد: قوالب الإيميل HTML.
الخاصية القديمة (Legacy) يُكتب على الوظيفة البديل الحديث (CSS) HTML5
align="left|center|right" <td>, <th>, <tr> محاذاة أفقية للمحتوى text-align: left|center|right; ملغى
valign="top|middle|bottom" <td>, <th>, <tr> محاذاة عمودية للمحتوى vertical-align: top|middle|bottom; ملغى
cellpadding="N" <table> الحشو الداخلي للخلايا td, th { padding: Npx; } ملغى
cellspacing="N" <table> المسافة بين الخلايا table { border-spacing: Npx; } ملغى
border="N" <table> سمك الإطار td, th { border: 1px solid #ccc; } قديم
bgcolor="color" <table>, <td> لون الخلفية background-color: color; ملغى
width="N" <table>, <td> تحديد العرض width: Npx; + table-layout: fixed قديم
height="N" <td>, <tr> تحديد الارتفاع height: Npx; قديم
summary="text" <table> وصف للشاشة (A11y) <caption> أو aria-describedby ملغى

١٢. السياق التاريخي لجداول HTML

من أداة عرض البيانات إلى أداة تخطيط الصفحات... وعودة

1993 – HTML الأولى
أُضيف وسم <table> في إحدى مسودات HTML لعرض البيانات العلمية والإحصائية فقط، وهو الغرض الأصلي الصحيح له.
1995 – 2000: عصر التخطيط بالجداول
مع صعود شعبية الويب وضعف دعم المتصفحات لـ CSS، اكتشف المطورون أن الجداول يمكنها إنشاء تخطيطات متعددة الأعمدة. أصبحت الجداول المتداخلة الطريقة الوحيدة المتاحة لبناء صفحات ذات تصميم معقد. كانت الصفحات تحتوي على عشرات الجداول المتداخلة!
2000 – 2007: ثورة CSS
مع تحسن دعم CSS في المتصفحات، بدأت حركة "CSS-based layouts" (Jeffrey Zeldman). تخلّى المطورون تدريجياً عن الجداول لصالح float وتخطيطات CSS النقية. أصبح استخدام الجداول للتخطيط يُعدّ ممارسة سيئة رسمياً.
2009 – 2015: CSS3 وFlexbox
CSS3 Flexbox أنهى أي مبرر لاستخدام الجداول في التخطيط. HTML5 ألغى رسمياً معظم خصائص الجداول التنسيقية. الجداول عادت إلى دورها الأصلي: عرض البيانات فقط.
2020 – حتى اليوم
CSS Grid وFlexbox هما المعيار الكامل للتخطيط. الجداول تُستخدم فقط للبيانات المجدولة. الاستثناء الوحيد المقبول لاستخدام جداول للتخطيط: قوالب الإيميل HTML، التي لا تزال محدودة دعم CSS.

١٦. جدول المرجع الشامل – Cheat Sheet

كل عناصر وخصائص HTML Tables في مكان واحد

العنصر / الخاصية يُكتب على الوصف HTML5
<table> — الحاوية الرئيسية للجدول. كل شيء يقع داخلها. ✅ مدعوم
<tr> داخل table أو thead أو tbody أو tfoot يُعرّف صفاً أفقياً واحداً. ✅ مدعوم
<td> داخل tr خلية بيانات عادية. الافتراضي: محاذاة يسار + وسط عمودي. ✅ مدعوم
<th> داخل tr خلية عنوان. الافتراضي: خط عريض + محاذاة وسط. ✅ مدعوم
<thead> داخل table قسم رأس الجدول. يحتوي عادةً على صف أو أكثر من th. ✅ مدعوم
<tbody> داخل table جسم الجدول. يمكن تعدده لتقسيم البيانات. ✅ مدعوم
<tfoot> داخل table تذييل الجدول. للمجاميع والملخصات. يُكتب آخراً في HTML5. ✅ مدعوم
<caption> أول عنصر داخل table عنوان مرئي للجدول. مهم جداً لإمكانية الوصول (A11y). ✅ مدعوم
<colgroup> بعد table مباشرة حاوية لتجميع وتنسيق الأعمدة. ✅ مدعوم
<col> داخل colgroup يُعرّف عموداً أو مجموعة أعمدة (عبر span) لتنسيقها. ✅ مدعوم
colspan="N" td, th يدمج الخلية أفقياً لتحتل مساحة N عمود. احذف N−1 خلية من نفس الصف. ✅ مدعوم
rowspan="N" td, th يدمج الخلية عمودياً لتمتد على N صف. احذف N−1 خلية من الصفوف التالية. ✅ مدعوم
scope="col|row|..." th يحدد اتجاه العنوان لقارئات الشاشة. مهم جداً لـ A11y. ✅ مدعوم – A11y
span="N" col يُطبّق تنسيق col الواحد على N عمود متتالية. ✅ مدعوم
cellpadding="N" table الحشو الداخلي للخلايا. ← CSS: padding: Npx على td ❌ ملغى
cellspacing="N" table المسافة بين الخلايا. ← CSS: border-spacing: Npx على table ❌ ملغى
align="..." td, th, tr محاذاة أفقية. ← CSS: text-align ❌ ملغى
valign="..." td, th, tr محاذاة عمودية. ← CSS: vertical-align ❌ ملغى
bgcolor="..." table, td لون الخلفية. ← CSS: background-color ❌ ملغى
summary="..." table وصف للجدول لقارئ الشاشة. ← استخدم <caption> ❌ ملغى
border="N" table سمك الإطار. ← CSS: border + border-collapse ⚠️ قديم
