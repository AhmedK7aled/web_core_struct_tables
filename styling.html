٥. محاذاة محتوى الخلايا

align وvalign القديمتين مقابل CSS الحديث

↔️ المحاذاة الأفقية (Horizontal)

قديماً: align="left|center|right" على الخلية مباشرة.


حديثاً (CSS): text-align: left | center | right;


📧
خاصية align القديمة لا تزال مستخدمة في قوالب الإيميل HTML بسبب محدودية دعم CSS فيها.

↕️ المحاذاة العمودية (Vertical)

قديماً: valign="top|middle|bottom" على الخلية مباشرة.


حديثاً (CSS): vertical-align: top | middle | bottom;


💡
<td> افتراضياً: يسار أفقياً + وسط عمودياً. <th> افتراضياً: وسط أفقياً + وسط عمودياً.

🏺 الطريقة القديمة (Legacy HTML)

<td align="right"
    valign="top"
    height="80">
  أعلى اليمين
</td>

✅ الطريقة الحديثة (CSS)

<td style="
  text-align: right;
  vertical-align: top;
  height: 80px;">
  أعلى اليمين
</td>

٦. المسافات والأبعاد

cellpadding, cellspacing, width, height – قديماً وحديثاً

📏 الفرق بين cellpadding وcellspacing

  • cellpadding — الحشو الداخلي: المسافة داخل الخلية بين المحتوى وحدودها. كمثل الـ padding في CSS. قديماً: <table cellpadding="10">. حديثاً: td, th { padding: 10px; }

  • cellspacing — المسافة الخارجية: المسافة بين الخلايا وبعضها البعض. قديماً: <table cellspacing="5">. حديثاً: table { border-spacing: 5px; }

  • border-collapse: collapse — لإزالة المسافات بين الخلايا كلياً ودمج الحدود في خط واحد.

📦 width & height – التحكم في الأبعاد

الجداول بطبيعتها مرنة (Flexible) وتتكيف مع حجم المحتوى تلقائياً، مما يجعل السيطرة عليها صعبة. لإجبار الجدول على أبعاد محددة:


  • تحديد عرض الخلية width يُطبَّق على العمود بأكمله.
  • تحديد ارتفاع الخلية height يُطبَّق على الصف بأكمله.
  • لضبط دقيق لعرض الأعمدة استخدم table-layout: fixed مع width على <col> أو <th>.

🏺 الطريقة القديمة

<table cellpadding="10"
       cellspacing="5"
       border="1">
  <tr>
    <td width="200"
        height="50">
      بيانات
    </td>
  </tr>
</table>

✅ الطريقة الحديثة (CSS)

table {
  border-spacing: 5px;
  border-collapse: separate;
}
td, th {
  padding: 10px;
  width: 200px;
  height: 50px;
  border: 1px solid #ccc;
}

٧. تجميع وتنسيق الأعمدة – <colgroup> & <col>

تنسيق أعمدة بأكملها بدلاً من تكرار الـ style على كل خلية

🎯 لماذا colgroup؟

بدلاً من تكرار كتابة class="highlight-col" على كل <td> في عمود كامل، يمكنك استخدام <colgroup> و<col> لتنسيق العمود مرة واحدة فقط. يجب وضع هذا الوسم مباشرة بعد فتح <table> (أو بعد <caption> إن وجد).


خاصية span تتيح لـ <col> واحد تطبيق تنسيقه على عدة أعمدة متتالية معاً.

colgroup + col + span 
<table border="1">
  <colgroup>
    <!-- العمود الأول – خلفية صفراء -->
    <col style="background-color: #fef08a; width: 120px;">
    <!-- العمودان التاليان – خلفية خضراء فاتحة -->
    <col span="2" style="background-color: #bbf7d0;">
  </colgroup>
  <thead>
    <tr>
      <th>المنتج</th>
      <th>الربع 1</th>
      <th>الربع 2</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>تفاح</td>
      <td>50</td>
      <td>60</td>
    </tr>
    <tr>
      <td>موز</td>
      <td>30</td>
      <td>45</td>
    </tr>
  </tbody>
</table>

٨. Modern UI – تحسينات واجهة المستخدم

table-layout: fixed, zebra striping, hover effects, sticky header

🎨 تقنيات الجداول الاحترافية

  • table-layout: fixed — تُجبر الجدول على احترام عرض الأعمدة المحدد بدقة، متجاهلةً طول المحتوى. ضروري للجداول ذات الأعمدة المحددة العرض.
  • Zebra Striping (تلوين متبادل) — تلوين صف وترك صف لتسهيل تتبع البيانات. يُستخدم CSS: tbody tr:nth-child(even) { background: #f8fafc; }
  • Hover Effect — إضاءة الصف عند مرور المؤشر لتحسين تجربة المستخدم: tr:hover { background: #e2e8f0; }
  • Sticky Header — تثبيت رأس الجدول أثناء التمرير: thead th { position: sticky; top: 0; }
CSS – نمط الجدول الاحترافي 
.modern-table {
  width: 100%;
  border-collapse: collapse;
  table-layout: fixed;
  border-radius: 8px;
  overflow: hidden;
}

.modern-table thead th {
  background: #1a3a5c;
  color: #00c8ff;
  padding: 12px;
  position: sticky;
  top: 0;
}

.modern-table tbody tr:nth-child(even) {
  background: #f0f5fa;
}

.modern-table tbody tr:hover {
  background: #d4e5f5;
  transition: background .15s;
}
