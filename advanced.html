٩. الجداول المتداخلة – Nested Tables

جدول داخل خلية جدول – ممارسة تاريخية يُنصح بتجنبها اليوم

🏺
السياق التاريخي: في أواخر التسعينيات وبداية الألفية الثالثة، كان المطورون يستخدمون الجداول المتداخلة لبناء تخطيطات الصفحات الكاملة. اليوم، يُستبدل هذا الأسلوب كلياً بـ CSS Flexbox وCSS Grid. الجداول المتداخلة لا تزال صالحة لأغراض عرض البيانات الحقيقية فقط.

📦 متى تستخدم الجداول المتداخلة؟

  • ✅ عرض بيانات معقدة ومتعددة المستويات (مثل جدول بيانات مضمّنة داخل خلية).
  • ✅ قوالب الإيميل HTML التي لا تدعم Flexbox بالكامل.
  • ❌ تصميم تخطيط الصفحة (استخدم CSS Grid/Flexbox).
  • ❌ أي غرض آخر غير عرض بيانات فعلية.
جدول متداخل – مثال 
<table border="1">
  <tr>
    <td>خلية خارجية</td>
    <td>
      <!-- جدول مستقل بالكامل داخل هذه الخلية -->
      <table border="1">
        <tr>
          <td>أ</td>
          <td>ب</td>
        </tr>
        <tr>
          <td>ج</td>
          <td>د</td>
        </tr>
      </table>
    </td>
  </tr>
</table>

١٠. إمكانية الوصول – Accessibility (A11y)

جعل الجداول مفهومة لقارئات الشاشة ومستخدمي لوحة المفاتيح

🔖 خاصية scope

تُضاف على <th> لإخبار قارئ الشاشة بإيجاه العنوان:


  • scope="col" — هذا العنوان يصف عمود (رأسياً).
  • scope="row" — هذا العنوان يصف صف (أفقياً).
  • scope="colgroup" — يصف مجموعة أعمدة.
  • scope="rowgroup" — يصف مجموعة صفوف.

📝 <caption>

يُعطي عنواناً مرئياً للجدول ويُقرأه قارئ الشاشة قبل محتوى الجدول. يُوضع مباشرة داخل <table> كأول عنصر. يُغني عن خاصية summary الملغاة في HTML5.


🚫
summary ملغى في HTML5. استخدم <caption> بدلاً منه.

🎯 كيف تضرّ الجداول السيئة بمستخدمي قارئات الشاشة؟

  • بدون scope، لا يعرف قارئ الشاشة أي عنوان يتبع أي خلية في جداول البيانات المعقدة.
  • الجداول المستخدمة للتخطيط تُشوّش تسلسل القراءة المنطقي، مما يُربك المستخدمين المكفوفين تماماً.
  • غياب <caption> يعني أن المستخدم لا يعرف ما يصفه الجدول قبل الدخول فيه.
جدول متوافق بالكامل مع معايير A11y 
<table>
  <caption>بيانات الطلاب – الفصل الأول</caption>

  <thead>
    <tr>
      <th scope="col">الاسم</th>
      <th scope="col">الدرجة</th>
      <th scope="col">التقدير</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <th scope="row">أحمد محمد</th>
      <td>90</td>
      <td>ممتاز</td>
    </tr>
  </tbody>
</table>

١٤. الجداول المتجاوبة – Responsive Tables

جعل الجداول تعمل بشكل صحيح على الشاشات الصغيرة والهواتف

📱
مشكلة الجداول على الهواتف: الجداول ذات الأعمدة الكثيرة تتجاوز عرض الشاشة على الهواتف وتسبب تمريراً أفقياً مزعجاً. الحل الأساسي هو لف الجدول داخل حاوية قابلة للتمرير أو إعادة هيكلة العرض بـ CSS.

🔧 الحل الأساسي – Overflow Wrapper

أبسط حل وأكثرها شيوعاً: لف الجدول في <div> مع overflow-x: auto. هذا يُتيح التمرير الأفقي فقط داخل حاوية الجدول دون كسر تخطيط باقي الصفحة.


💡
أضف min-width على الجدول نفسه لضمان ظهور كل الأعمدة بشكل صحيح بدون تشويه.

🎯 table-layout: fixed مع overflow

مجموعة خصائص CSS الثلاث الأساسية لجدول متجاوب احترافي:


  • overflow-x: auto على الحاوية — تمرير أفقي فقط عند الحاجة
  • table-layout: fixed على الجدول — أعمدة ثابتة العرض لا تتغير بالمحتوى
  • text-overflow: ellipsis على الخلايا — اقتطاع النصوص الطويلة بنقاط

❌ بدون تجاوب – ينكسر على الهاتف

<!-- لا حاوية – يتجاوز الشاشة -->
<table border="1">
  <tr>
    <th>عمود 1</th>
    <th>عمود 2</th>
    <th>عمود 3</th>
    <th>عمود 4</th>
    <th>عمود 5</th>
  </tr>
</table>

✅ مع تجاوب – يعمل على كل الشاشات

<!-- حاوية مع overflow -->
<div style="overflow-x:auto;">
  <table style="min-width:500px;">
    <tr>
      <th>عمود 1</th>
      <th>عمود 2</th>
      <th>عمود 3</th>
      <th>عمود 4</th>
      <th>عمود 5</th>
    </tr>
  </table>
</div>
CSS – نظام الجدول المتجاوب الكامل 
/* الحاوية – تتيح التمرير الأفقي عند الحاجة */
.table-wrapper {
  overflow-x: auto;
  -webkit-overflow-scrolling: touch; /* سلاسة على iOS */
  border-radius: 8px;
  border: 1px solid #e2e8f0;
}

.responsive-table {
  width: 100%;
  min-width: 600px; /* يمنع الجدول من الانضغاط */
  border-collapse: collapse;
  table-layout: fixed;
}

.responsive-table td,
.responsive-table th {
  padding: 12px 16px;
  text-align: right;
  white-space: nowrap; /* يمنع كسر الخلايا */
  overflow: hidden;
  text-overflow: ellipsis; /* ... عند الاقتطاع */
}

/* على الشاشات الصغيرة – إخفاء أعمدة غير ضرورية */
@media (max-width: 480px) {
  .hide-mobile { display: none; }
}
